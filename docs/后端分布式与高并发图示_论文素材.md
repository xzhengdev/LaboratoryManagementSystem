## 图1 请求到汇总的端到端时序图

```mermaid
sequenceDiagram
    autonumber
    participant Client as 前端(PC/小程序)
    participant API as Flask API
    participant Auth as 鉴权/权限
    participant Router as 分库路由(db_router_service)
    participant ShardDB as 校区库(lab_campus_x)
    participant FS as 校区SeaweedFS
    participant Redis as Redis(队列/锁/限流)
    participant Worker as event_worker
    participant Summary as 汇总库(lab_summary)

    Client->>API: 发起业务请求(日报/资产/头像上传)
    API->>Auth: JWT校验 + 角色校验
    Auth-->>API: 通过

    API->>Redis: 限流检查(可选)
    API->>Redis: 幂等键检查(可选)

    API->>Router: 按 campus_id 选择分库
    Router-->>API: 返回校区会话
    API->>ShardDB: 写入业务主数据

    alt 包含图片上传
        API->>FS: 按 campus_id 上传到对应SeaweedFS
        FS-->>API: 返回 file_id/url
        API->>ShardDB: 写入 file_objects 元数据
        API->>ShardDB: 回写业务字段(如 users.avatar_url)
    end

    API->>Redis: 发布异步事件(lab:events)
    API-->>Client: 快速返回成功

    Worker->>Redis: 消费事件
    Worker->>Router: 读取对应校区库数据
    Worker->>Summary: 增量汇总写入 campus_summary_snapshots
```

**建议图题**：图4-X 分布式请求处理与异步汇总时序图

## 后续画图模板（写论文时直接套用）

### 1. 业务时序图模板（适合日报、资产、审批）

```mermaid
sequenceDiagram
    autonumber
    participant U as 用户
    participant API as API
    participant R as 分库路由
    participant DB as 校区库
    participant MQ as Redis队列
    participant W as Worker
    participant S as 汇总库

    U->>API: 提交业务请求
    API->>R: 解析 campus_id
    R-->>API: 返回目标分库
    API->>DB: 事务写入
    API->>MQ: 发布事件
    API-->>U: 返回成功
    W->>MQ: 消费事件
    W->>S: 更新汇总快照
```

## 使用提醒（后续写论文务必保留）

1. 每一章至少放 1 张“结构图或流程图”，避免纯文字。  
2. 分布式章节优先放“分库路由图 + 多校区部署图”。  
3. 高并发内容放“限流/幂等/异步流程时序图”，穿插在实现处。  
4. 图题统一格式：`图X-Y 名称`，并在正文中先引用再贴图。  
5. 图下方可加 1~2 句说明“本图体现的分布式/并发落点”。
