# Agent v2 评测用例清单与对话脚本模板

## 1. 评测目标

验证 Agent 是否实现“规划-工具调用-再规划-完成任务”的闭环能力，覆盖成功路径、缺参路径、异常路径和权限路径。

## 2. 评测用例清单

### A. 正常成功路径

1. 查实验室
- 输入：`明天有哪些可预约的实验室？`
- 预期：调用 `query_labs`，返回实验室列表。

2. 查排期
- 输入：`帮我看 2026-04-20 的排期`
- 预期：调用 `query_schedule`，返回排期摘要。

3. 推荐时间
- 输入：`给我推荐明天下午可用时间`
- 预期：调用 `recommend_time`，返回推荐时段。

4. 一句话完成预约
- 输入：`帮我预约明天9:00-11:00，3人，用于课程实验`
- 预期：循环调用 `query_labs/query_schedule/recommend_time/create_reservation` 中必要步骤，最终预约成功。

5. 查看我的预约
- 输入：`查看我的预约`
- 预期：调用 `my_reservations` 并返回记录。

6. 取消预约
- 输入：`取消预约ID 12`
- 预期：调用 `cancel_reservation`，成功则返回已取消。

### B. 缺参与追问路径

7. 预约缺日期
- 输入：`帮我预约实验室`
- 预期：返回缺失信息提示（日期/时间/人数/用途）。

8. 查排期缺日期
- 输入：`帮我查排期`
- 预期：返回“请先告诉我日期”。

9. 取消预约缺ID
- 输入：`帮我取消一个预约`
- 预期：返回“请提供预约ID”。

### C. 业务规则与失败路径

10. 人数超容量
- 输入：`明天9:00-11:00预约A101，200人，用于上课`
- 预期：`create_reservation` 失败并返回可解释业务错误。

11. 时间冲突
- 前置：该用户已有重叠预约。
- 输入：`再帮我约同一时间段`
- 预期：返回冲突提示。

12. 非法时间格式
- 输入：`帮我预约明天上午九点半到十一点半半`
- 预期：无法解析时返回格式提示。

13. 不存在实验室ID
- 输入：`帮我预约实验室ID 9999，明天9:00-11:00，2人，用于实验`
- 预期：返回“未找到对应实验室”。

14. 不存在预约ID取消
- 输入：`取消预约ID 99999`
- 预期：返回“没有找到预约ID”。

### D. 兜底与鲁棒性

15. 非实验室闲聊
- 输入：`今天心情不好怎么办`
- 预期：走通用对话，不触发实验室工具。

16. 退出当前预约流程
- 输入：`不需要预约了`
- 预期：清空当前预约表单状态。

17. 重复工具保护
- 输入：构造让 Planner 连续重复同工具的请求。
- 预期：触发重复上限保护并提示用户补充信息。

18. LLM 不可用回退
- 前置：移除 `LLM_API_KEY`。
- 输入：`帮我查明天空闲实验室`
- 预期：规则引擎仍可处理。

## 3. 对话脚本模板（可复用）

### 模板 A：预约闭环

1. 用户：`帮我预约明天上午实验室`
2. Agent：`请补充开始时间、结束时间、人数、用途。`
3. 用户：`9:00-11:00，3人，用于课程实验`
4. Agent：`先筛选实验室 -> 查排期 -> 创建预约`
5. Agent：`已帮你完成预约: ...`

验收点：

1. 至少出现一次“工具执行->再规划”。
2. 最终成功创建预约。

### 模板 B：取消预约

1. 用户：`查看我的预约`
2. Agent：返回预约列表（含 ID）
3. 用户：`取消预约ID 23`
4. Agent：返回取消成功结果

验收点：

1. `my_reservations` 后可直接执行 `cancel_reservation`。
2. 返回结果具备可解释性。

### 模板 C：失败可恢复

1. 用户：`帮我预约今天过去的时间段`
2. Agent：返回失败原因（业务规则）
3. 用户：`那改成明天9:00-11:00，2人，用于实验`
4. Agent：继续流程并成功预约

验收点：

1. 失败后不会“卡死”。
2. 新输入可继续推进并完成任务。

## 4. 记录模板（建议验收表）

每条用例建议记录：

1. 用例编号
2. 输入文本
3. 实际触发工具序列
4. 最终回复
5. 是否通过
6. 备注（错误码/改进建议）
