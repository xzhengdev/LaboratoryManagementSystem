# 分库开发启动指南

适用场景：
- 论文和系统都要体现“每个校区独立数据库 + 中心汇总”
- 后续要联调预约、资产、日报、统计的分库版本

## 1. 先配置环境变量

编辑 `backend/.env`：

```env
ENABLE_CAMPUS_DB_ROUTING=1
CAMPUS_DB_URI_MAP={"1":"mysql+pymysql://root:password@127.0.0.1:3306/lab_campus_1?charset=utf8mb4","2":"mysql+pymysql://root:password@127.0.0.1:3306/lab_campus_2?charset=utf8mb4"}
SUMMARY_DB_URL=mysql+pymysql://root:password@127.0.0.1:3306/lab_summary?charset=utf8mb4
```

说明：
- `CAMPUS_DB_URI_MAP` 的 key 是 `campus_id`
- value 是该校区数据库连接串
- `SUMMARY_DB_URL` 是中心汇总库

## 2. 初始化分库结构

在 `backend/` 目录执行：

```bash
python scripts/bootstrap_shards.py
```

该脚本会：
- 读取 `CAMPUS_DB_URI_MAP`
- 在每个校区库创建当前模型对应表结构
- 如果配置了 `SUMMARY_DB_URL`，也会初始化中心库

## 3. 做分库自检

在 `backend/` 目录执行：

```bash
python scripts/check_shards.py
```

该脚本会检查：
- 每个库关键表是否存在
- 预约、资产申报、日报记录数输出

## 4. 再启动后端联调

```bash
python run.py
```

## 5. 联调建议顺序

1. 预约创建/审批（验证按校区入库）
2. 资产申报/审批/入库（验证预算锁定）
3. 日报图片上传/提交/审核（验证文件元数据与校区绑定）
4. 统计接口（验证跨校区聚合）
