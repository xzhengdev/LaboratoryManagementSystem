**表4-3 并发控制措施表**

| 措施 | 使用位置 | 解决的问题 |
|---|---|---|
| 幂等键 | 提交类接口入口 | 防止重复点击或网络重试导致重复写入 |
| 分布式锁 | 预约时段、预算额度等热点资源更新前 | 防止多个请求同时修改同一资源 |
| 数据库事务 | 预算、审批、入库等关键写操作 | 保证相关数据同时成功或同时失败 |
| 缓存 | 实验室列表、设备列表、统计查询等高频读接口 | 减少数据库访问压力 |
| 限流 | 网关或接口层 | 防止短时间大量请求冲击后端服务 |
| 异步事件 | 汇总统计、日志、通知等非核心流程 | 缩短用户等待时间，降低主流程压力 |
